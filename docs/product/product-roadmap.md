## 2. MVP / V1.0 版本

### 2.1 核心目标

**上线时间**: 4 周
**核心价值**: 验证 AI 对话能力，实现候选人咨询到面试预约的完整流程

### 2.2 功能清单（P0 优先级）

| 功能模块 | 功能点 | 技术实现 | 验收标准 |
|---------|--------|---------|---------|
| **消息接收** | 企微回调处理 | MessageController + 消息队列 | 100ms 内响应企微服务器 |
| **智能对话** | 主动欢迎语 | AgentService + 预设提示词 | 添加好友后 3 秒内发送 |
| | 岗位咨询（多轮） | Claude API + 上下文管理 | 支持 5+ 轮连续对话 |
| | 面试安排 | 渐进式信息收集 | 自动收集姓名/电话/年龄 |
| **消息发送** | 文本消息发送 | WeCom API | 发送成功率 > 99% |
| | 消息分段（长文本） | 智能分段算法 | 单段 < 500 字 |
| **基础设施** | 日志记录 | Winston + 结构化日志 | 全流程可追溯 |
| | 环境配置 | dotenv + 配置验证 | 启动前自动检查 |

### 2.3 技术要点

- ✅ 消息去重（5 分钟窗口）
- ✅ 消息历史管理（最近 20 条）
- ✅ 智能消息聚合（1 秒窗口，最多 3 条）
- ✅ Agent 缓存策略（纯文本对话缓存 1 小时）
- ✅ 错误重试机制（指数退避，最多 3 次）

### 2.4 验收标准

- [ ] 候选人从咨询到预约面试，全流程自动化
- [ ] 平均响应时间 < 5 秒
- [ ] AI 回答准确率 > 85%
- [ ] 系统稳定性 > 99%（7x24 小时运行）

## 3. V1.1 增强版

### 3.1 核心目标

**上线时间**: V1.0 后 2 周
**核心价值**: 降低面试爽约率，提升到面率

### 3.2 功能清单（P1 优先级）

| 功能模块 | 功能点 | 实现方式 | 预期效果 |
|---------|--------|---------|---------|
| **定时任务** | 面试提醒（提前 1 天） | Bull Queue + Cron | 爽约率降低 30% |
| | 面试提醒（当天） | 定时任务 | 到面率提升 20% |
| **主动跟进** | 面试后第二天跟进 | AI 主动发起对话 | 试工安排率 > 60% |
| | 签约进度跟进 | 定时检查 + 主动提醒 | 入职转化率 > 80% |
| **店长协同** | 每日面试名单推送 | 定时任务 + 格式化消息 | 店长接收及时率 100% |

### 3.3 技术要点

- ✅ 定时任务调度（Bull Queue）
- ✅ 面试状态追踪（Redis 缓存）
- ✅ 主动消息发送（非回调触发）
- ✅ 消息模板管理

## 5. 技术风险与应对

### 5.1 关键风险

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|---------|
| **Agent API 不稳定** | 对话中断 | 中 | 1. 多次重试机制<br>2. 降级策略（预设回复）<br>3. 监控告警 |
| **企微回调超时** | 消息丢失 | 低 | 1. 异步处理（< 100ms 响应）<br>2. 消息队列缓冲<br>3. 失败重发 |
| **AI 回答不准确** | 用户体验差 | 中 | 1. 持续优化 System Prompt<br>2. 人工审核反馈<br>3. 知识库补充 |
| **高并发消息** | 系统过载 | 中 | 1. 消息队列削峰<br>2. Redis 缓存<br>3. 水平扩展 |
| **隐私数据泄露** | 合规风险 | 低 | 1. 数据加密存储<br>2. 敏感信息脱敏<br>3. 访问权限控制 |

### 5.2 性能保障

- **消息处理能力**: 1000+ 条/小时
- **并发对话数**: 100+ 个
- **系统可用性**: 99.9%
- **灾难恢复**: 1 小时内恢复
